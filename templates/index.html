              Quick Help Guide

              Understanding the Symbols:

              ‚úîÔ∏è: Confirmed "Yes" (e.g., paper is a Survey, uses SMT, etc.);

              ‚ùå: Confirmed "No";

              ‚ùî: Unknown / Not Classified / Not Applicable;

              üë§: Last changed or verified by the (hopefully human) user;

              üñ•Ô∏è: Last changed or verified by an AI Model (hover for model name);

              How to Use:

              Filter Papers: Use the year range, minimum page count, and the search bar at the top. Press "Apply" (appears on change) for server-side filters.

              Toggle Columns: Use checkboxes like "Hide Offtopic", "Hide X-Ray", or "Only Survey" for quick client-side filtering. As the checkboxes have no permanent labels, hover each checkbox to identify its purpose.

              Classify/Verify: Click the "Classify this paper" or "Verify this paper" button in the detail row to run AI on a single paper. Use the top buttons for batch processing.

              Edit Data: Click on any ‚úîÔ∏è / ‚ùå / ‚ùî symbol in the main table to cycle its value (this is automatically saved on change). Click 'show' in the details column to edit text fields (like "Model Name" or "Other Defects"). These have to be saved manually by clicking on "Save Changes". Editing functionality is unavailable on HTML exports.

              Search: Type on the search bar for instant search on all relevant fields. User comments are searchable. Classifier/Verifier traces are not (to avoid false-positives/noise)

              View Statistics: Click the "View Statistics" button to see charts and lists of repeating journals, authors, keywords, and mentioned features/models as well as relevant charts. The stats are calculated based on currently visible filtered data.

              Export Data: Use the "Export HTML" or "Export XLSX" buttons to download the currently filtered dataset.

              Import Data: Use the "Import BibTeX" button to add new papers to the database from a .bib file. Import/Export functionality is unavailable on HTML exports.

              Tip: The table is pre-sorted to show papers with user comments first. You can click any column header to sort by that field.

              About

              PCB Inspection Papers Database Browser - bibtex-custom-parser

              Developed by Luis Alfredo da Silva, to assist his own Master's degree research at Universidade do Estado de Santa Catarina (UDESC).

              The following open source libraries, and its dependencies, are leveraged by this software:

              Backend: Python, Flask, SQLite3

              Frontend: HTML5, CSS3, JavaScript (Vanilla JS)

              Data Visualization: Chart.js

              Utilities: rcssmin, rjsmin (minification), openpyxl (Excel export), pako (compressed HTML export)

              Classification and verification was inferenced by Unsloth's Dynamic quants of Qwen3-30B-A3B-Thinking-2507 model from Qwen/Alibaba Cloud.

              Local Inference engine: llama.cpp.

              Original papers data (BibTeX) was sourced from Scopus, IEEE Xplore and ACM Digital Library
